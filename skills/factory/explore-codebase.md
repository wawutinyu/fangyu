# explore-codebase

先摸清地图，再下结论。禁止臆造路径与 API。

## 步骤

1. **定范围**：用 `glob` / `list` 找入口、包结构、相关目录。
2. **搜符号**：用 `grep` / `search` 定位函数、类、配置键、错误文案。
3. **精读**：对命中文件用 `read`；大文件先读头尾与关键函数。
4. **交叉验证**：至少两条证据（路径 + 符号 / 调用方）再下结论。
5. **收口**：结论含具体路径、符号名、一两句机制说明；不要改文件。

## 委派

父 Agent 可用 `task` → `explore` 做隔离探索；本技能正文也可由主环 `skill_load` 加载。

## 反例

- 没搜就声称「没有这个模块」
- 只根据文件名猜测实现
- 探索过程中 `write` / `shell` 改仓
