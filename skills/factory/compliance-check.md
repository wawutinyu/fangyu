# compliance-check

按清单打勾，缺口要可复现；禁止口头「大概都合规」。

## 步骤

1. **锁定清单**：来源（内部政策路径或标准清单）；版本/日期写清。
2. **逐项取证**：每项至少一条证据（配置路径、代码符号、文档段落、截图路径）。
3. **状态**：通过 / 缺口 / 不适用；缺口写清修复建议与责任面（工程/运营/法务）。
4. **汇总**：高风险缺口置顶；输出 `write_deliverable` → `deliverables/compliance/`。
5. **收口**：列出未覆盖范围，避免假装全检。

## 反例

- 清单空着却写「已合规」
- 用过时政策版本核对
- 把「计划整改」写成「已通过」
