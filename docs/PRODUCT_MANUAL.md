# 方隅（fangyu）产品说明书

> 这是「产品是什么」——给决策者、新同事、以及第一次听说方隅的人。  
> **怎么点开、怎么操作** → 见 [使用手册](USER_GUIDE.md)。

*版本：2026-07-19*

---

## 1. 一句话

**方隅是 Agent 工厂，不是又一个聊天窗口。**

你在可视化界面里设计「能干活的智能体」，导出成可独立运行的包（Bundle），带着约束（宪法）和工具离开 Studio，还能和其他 Agent / 工厂组网协作。

口诀：

> **MCP 连世界，Skills 装方法，A2A 连同伴，Harness 当机床，Eval 做出厂质检；观是现场。**

---

## 2. 它解决什么问题

| 常见痛点 | 方隅怎么对 |
|----------|------------|
| 聊天 Bot 只能问答，不能稳定改文件/跑命令 | 机床 + 工具带 + 工作区约束 |
| Agent 只能活在某个 SaaS 里，带走不了 | **导出 Bundle**，离 Studio 仍可跑 |
| 多 Agent / 多机协作只剩 JSON 日志 | **观** 值班现场 + A2A 通讯录 |
| 能力没有边界 | **律** 宪法：禁什么、审计什么 |
| 「演示过」≠「可出厂」 | **Eval / 门禁** 脚本可重复验收 |

---

## 3. 它不是什么

- **不是** ChatGPT / Cursor 的替代品（日常对话、写代码 IDE）  
- **不是** 纯 workflow 工具（n8n / Dify）——方隅强调 **可导出公民**，不是只在平台里跑  
- **不是** 下载即用的零安装消费级 App（当前仍偏工程向；桌面壳 = 同一套 UI + 本机执行端）  
- **不是**「官方送你一个 OpenCode」——平台能力要用 **节点编排自己拼**；整环节点只是可选捷径  

---

## 4. 产品结构：四门两包

> **序以设计，行以执行，观以察局，律以约束。**

| 门 | 你用它做什么 | 在哪里 |
|----|----------------|--------|
| **序** | 画流程 / 画 Agent 网、意图生成、预览、导出 | 浏览器 Studio 或桌面窗口（主界面） |
| **行** | 本机真执行（shell、文件、Worker 任务） | Worker / 托盘；预览也可走本机 API |
| **观** | 看多 Agent / 跨厂状态、告警、回放 | 顶栏「观」 |
| **律** | 宪法、禁止动作、信任与审计 | 顶栏「律」 |

两包交付：

| 包 | 内容 |
|----|------|
| **序包** | `fangyu-studio`（含观、律界面） |
| **行包** | `fangyu-worker`（本机执行） |

桌面原生（Tauri）= 序 UI + 自动拉起本机 API + Worker，**画面与网页同一套**，不是另一套产品。

## 5. 核心概念（读懂界面的词典）

| 词 | 意思 |
|----|------|
| **Flow 画布** | 流程图：节点 = 一步能力（输入、大模型、工具轮、记忆、循环…） |
| **Agent 画布** | 组织图：多个 Agent 谁调谁、怎么协作 |
| **节点** | 可拖拽的一步；拼起来才是完整智能体行为 |
| **工具轮 / 整环** | 工具轮 = 可拼的一轮「想→调工具→回灌」；整环 = 可选的多轮黑盒捷径 |
| **Bundle** | 导出的独立 Agent 包：配置、技能、宪法、工作区约定 |
| **工作区 workspace** | Agent 允许读写的目录边界 |
| **工厂 / profile** | 按模板批量出包（如办公编队、coding 等） |
| **A2A** | Agent 之间互相发现、调用的协议与通讯录 |
| **宪法** | 包内规则：禁止危险动作、要审计等 |
| **门禁 / Eval** | 自动检查：单测、导出材料、live 脚本等 |

## 6. 主要能力清单（当前）

### 已可用（可演示）

- Flow / Agent 双画布编排与预览  
- 意图生成（自然语言 → 流程或 Agent 网）  
- 原料：工具 / 技能 / MCP 货架  
- Bundle 导出与独立运行（`run-bundle` / 托管 manage）  
- 办公编队、多 Agent 编排、边 ACL（引擎侧）  
- 跨厂通讯录、心跳、双厂值班验收脚本  
- 观：值班、告警、时间轴  
- 律：宪法扫描与策略  
- 飞书通道代码与运维向导（**真机订阅暂缓**）  
- 出厂门禁 `factory_gate`、竖切 demo 脚本  

### 明确暂缓 / 偏弱

- 飞书 **真机** 私聊闭环（需你开口再开）  
- 非开发者「下载即用」零安装体验  
- 边 ACL 的画布可视化编辑仍薄  
- 长期 live 稳定性需持续盯  

更细的毕业进度见 [毕业标准](GRADUATION_EXPORTABLE_AGENT.md)、[项目评估](PROJECT_ASSESSMENT.md)。

## 7. 推荐应用场景（建议从这里用）

| 场景 | 你得到什么 | 入门方式 |
|------|------------|----------|
| **办公周报协作** | 多角色起草→审校→落盘 | 意图生成办公编队 / `demo_vertical_slice` |
| **绑目录改文件** | 在工作区内读改写 | 拼装验收（循环+工具轮）或 coding 出包 |
| **双厂值班** | 两个实例互探、告警可处置 | 运维·工厂 + 双厂脚本 |
| **本机 Worker 任务** | 行包跑 shell / Flow | 启动 Worker 后「派发至行」 |

场景剧本与验收：[可演示竖切](DEMO_VERTICAL_SLICE.md)、[双厂值班](DUAL_FACTORY_DUTY.md)、[拼装验收](PLATFORM_COMPOSE_HARNESS.md)。

## 8. 谁适合用

| 适合 | 不太适合（现阶段） |
|------|---------------------|
| 愿意本机起服务的开发者 / 小团队 | 只想装 App 点一下的纯小白 |
| 要「带走」Agent 的人 | 只要网页里聊两句的人 |
| 要多 Agent / 跨机协作与治理的人 | 只要单个聊天 Bot 的人 |

- 浏览器 Chrome / Edge / Safari
