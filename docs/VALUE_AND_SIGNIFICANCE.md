# 工作价值与意义（2026 雷达对齐）

> 把「做了什么」收成「为什么值得」——对照行业五层栈与方隅「Agent 工厂」定位。  
> 关联：[技术雷达](TECH_RADAR_2026.md) · [愿景](VISION_AND_PRODUCT.md) · [工厂原料](FACTORY_MATERIALS.md) · [出厂 Eval](FACTORY_EVAL.md) · [A2A 发现](A2A_DISCOVERY.md)

---

## 1. 总命题

行业已收敛：**MCP（工具）· Skills（方法）· A2A（协作）· Harness（机床）· Eval（质检）**。

方隅不是再做一个聊天壳，而是 **Agent 工厂**：原料可声明、机床可跑、协作可联邦、出厂可测、现场可值班。

本轮工作的意义，是把这五层从「概念对齐」推进到「产线可用」，并让人能在 **观** 上看见、管起来。

**口诀：MCP 连世界，Skills 装方法，A2A 连同伴，Harness 当机床，Eval 做出厂质检；观是现场。**

---

## 2. 三层递进（价值结构）

```text
① 原料层    工具（MCP）+ 方法（Skills）     → 工厂有货可装
② 现场层    观（Presence 可视化）           → 协作可看见、可回放
③ 运维层    A2A 跨厂 + Eval/观测闭环         → 联邦可值班、质检可回归
```

下层支撑上层；上层不替代下层。没有原料，跨厂是空壳 RPC；没有观，告警只是日志；没有 Eval，演示不等于可毕业。

## 3. 分块价值

### 3.1 工具（MCP）— 连世界

| 做了什么 | 价值 | 意义 |
|----------|------|------|
| 原料货架 · materials 进货 · toolbelt | 工具可勾选、可进 Bundle | **能声明才算进厂**，不是 Studio 里随手挂一个节点 |
| MCP `tools:*` · 无状态 HTTP · Tasks 子集 | 可插拔工具总线 | 对齐行业 MCP，而不是自造另一套 RPC |
| 浏览器 Computer-use · SSO/OIDC | 真实环境操作 + 身份 | 工厂公民能出门办事，且带身份约束 |

**意义一句话：** 把「模型会调工具」变成「工厂有标准原料带」，模型可换、协议与货架要稳。

### 3.2 方法（Skills）— 装方法

| 做了什么 | 价值 | 意义 |
|----------|------|------|
| Skills frontmatter · 按需加载 | 目录摘要进系统、全文渐进 | 对齐 SKILL.md 实践，控制上下文成本 |
| 技能包扩容（explore / research / office / legal…） | 方法可打包、可复用 | 工厂出的是「会做事的公民」，不是裸 loop |
| 技能目录 UX · 人审闸 | 可浏览、可约束危险动作 | 方法进产线，同时进治理 |

**意义一句话：** 方法与工具分层——Skills 不是又一个 API，而是可版本化的作业知识。

### 3.3 Harness（机床）— 执行系统

| 做了什么 | 价值 | 意义 |
|----------|------|------|
| plan / task / shell ask · 拓扑 depends | 可编排、可委派、可闸 | 「再换个更强模型」解决不了的稳定性问题 |
| harness_trace 落盘 | 执行可回看 | 观测与 Eval 有原料，不是黑盒 demo |

**意义一句话：** 机床先于场景炫技；IM/办公是产线，不是底座借口。

### 3.4 观（Presence）— 协作现场

| 做了什么 | 价值 | 意义 |
|----------|------|------|
| 宅院共场 · 角色/边/径 · SSE · 回放 | 多 Agent / 跨机状态可见 | L2 互操作有人机面，不是只剩 JSON |
| 托管/主机进观 · 值班大屏 | 运维与协作同一张图 | 「谁在忙、谁离线、谁在跨厂」一眼可读 |
| 白话解释 · 告警铃铛/条 · 去观定位 | 事件可理解、可跳转 | 观从「好看」变成「能值班」 |

**意义一句话：** 观是工厂的现场——没有现场，A2A 与 Eval 的告警落不到人身上。

### 3.5 A2A 跨厂 + Eval 观测 — 联邦可运营

| 做了什么 | 价值 | 意义 |
|----------|------|------|
| 发现 / 通讯录 / 心跳 / 对齐 | 对端可登记、可探活 | A2A 从「调通 RPC」到「有通讯录」 |
| 授权向导 · 试跑 · ACL · 无 Key 跨厂 smoke | 外部 Agent 可准入、可验 | 联邦有边界，不是裸连 |
| 健康分 · 离线/试跑/健康回归告警 | 质量与可达性可量化 | 值班有指标，不只靠感觉 |
| 一键/批量再探测 · 写观 · 时间轴筛选 | 告警可处置、可回看 | 闭环：发现 → 告警 → 处置 → 复盘 |
| Eval 报告/对比/趋势挂工厂健康 | 出厂质检带舰队快照 | 门禁不只测本机，还看通讯录健康 |

**意义一句话：** 把 A2A 做成可运营的跨厂产线，把 Eval 做成带回归的出厂质检——对齐「可毕业公民」，而不是偶发 demo。

## 4. 对「AI 社会」愿景的贡献

对照 [愿景](VISION_AND_PRODUCT.md) 三条承诺：

| 承诺 | 本轮贡献 |
|------|----------|
| **独立节点** | 原料 + Card + gate：导出物可声明、可测 |
| **相互协作** | A2A 发现/通讯录/跨厂 smoke + 观现场：网络可看见 |
| **共同约束** | 宪法/ACL/人审/授权向导：联邦有闸，不是纯能力堆砌 |

策略未变：**先机床与原料，再场景**；真 IM 仍可暂缓，不挡底座主航道。

---

## 5. 刻意不做 / 边界

- **真 IM（飞书真机）**：场景产线，雷达暂缓；需要时单开，不混进底座刀。  
- **再堆同类告警/筛选按钮**：边际收益递减；下一优先是 **真双厂验收** 或真 IM。  
- **「只换更强模型」**：不替代 MCP/Skills/A2A/Harness/Eval 分层。

---

## 6. 一句话收束

**工具与方法**让工厂有货可装；**观**让协作有现场可看；**跨厂 A2A + Eval**让联邦可值班、出厂可质检。  
合在一起，才是 2026 雷达下的方隅：不是聊天壳，而是能批量组装、可导出、可观测的 Agent 工厂。
